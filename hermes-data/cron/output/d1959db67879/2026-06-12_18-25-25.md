### 图片获取工作流（严格按顺序执行，不可跳步）

**每张图片必须从该板块对应的真实新闻源爬取og:image，不能「随便找张图凑合」。Kali原话：「我不是让你随便去下载，我是让你去找到这个源头的地方，把他们那个源头的配图拿过来，是让你去爬，用爬虫」。**

1. **web_search 搜具体新闻标题** → 拿到真实文章URL（如 `site:dexerto.com Wuthering Waves Cyberpunk Edgerunners`）
2. **urllib 抓 HTML 提取 og:image** → 正则匹配 `<meta property="og:image" content="...">`
3. **如果 urllib 被封（403/401/429）** → 用 Playwright 浏览器 `mcp_playwright_browser_navigate` + `evaluate` 提取 og:image URL
4. **如果 og:image 下载失败** → 换同一条新闻的另一个来源URL（Dexerto/PBS/Yahoo Finance等通常成功）
5. **如果所有新闻源都失败** → picsum.photos 语义化seed（如 `seed/wuthering-waves-cyberpunk/1200/675`）
6. **⛔ 绝对禁止用随机无关图片充数** — 宁可缺图也不能放不对的图
7. **下载后必须**：Pillow resize到≥1200px宽 + JPEG q85 + MD5唯一性检查
8. **特定板块必须用特定真实图片**：鸣潮×边缘行者用官方联名图，NASA用NASA官方图，Spacex用SpaceX图

```python
import hashlib, base64
seen = {}
for name, b64 in B.items():
    raw = base64.b64decode(b64)
    h = hashlib.md5(raw).hexdigest()
    if h in seen:
        print(f"⚠️ DUPLICATE: {name} = {seen[h]}")
    else:
        seen[h] = name
print(f"Images: {len(B)} total, {len(seen)} unique")
```

### ⛔ 铁律：HTML模板图片代码常见Bug

**Bug #1：文化卡片img()未调用（v7.0-v7.1实际bug）**
文化卡片的`img_h`变量直接用了`c["img"]`（key名字符串如`"tyshawn_sorey"`）而不是`img(c["img"])`（返回data URI）。结果：文化板块所有图片显示为空白。修复：
```python
# ❌ 错误（输出key名字符串）
img_h = f'<div class="cult-img"><img src="{c["img"]}" alt="{c["title"]}"></div>'
# ✅ 正确（调用img()函数输出data URI）
img_h = f'<div class="cult-img"><img src="{img(c["img"])}" alt="{c["title"]}"></div>'
```
详情页hero img同样问题：`{c["img"]}` → `{img(c["img"])}`。

**Bug #2：WebP图片MIME检测**
优化脚本跳过小文件（<200KB）不转JPEG时，WebP格式的base64以`UklGR`开头。`img()`函数必须检测`UklGR`前缀并加`data:image/webp;base64,`前缀，否则WebP图片无法显示。

**Bug #3：新闻源头og:image爬取流程**
必须从真实新闻URL爬取og:image，不能用picsum随机图充数。流程：
1. 用web_search搜每条新闻的真实文章URL
2. 用urllib抓取HTML，正则提取`<meta property="og:image" content="...">`
3. 下载图片并base64编码
4. 用Pillow压缩（>800px宽或>200KB的缩到800px JPEG q80）
5. picsum仅作为最后fallback，seed必须语义化（如`jazz-saxophone-music`）

**Bug #4：新增板块没有卡片+详情页交互（v7.2实际bug）**
Kali原话：「这几个板块也需要像其它板块那样，所有的都需要同步包括文章要求和图片要求」。经济、生命科学、基金等新板块必须：
- ✅ 使用跟核心信号/科研/文化完全一样的卡片格式（带图片、标签、标题、摘要）
- ✅ 每张卡片点击进入详情页（3-4章节深度分析，每章3-5句）
- ✅ 详情页段落间空一行（`<p style="margin-top:16px">`）
- ✅ 板块标题用`sec-head`格式（编号+标题）
- ✅ 图片必须跟板块内容对应
- ❌ 绝不能只放一个数据表格或纯文字列表就完事

## 常见失败模式

- **时间判定失败**：没检查当前时间就写内容。→ 必须第一件事查北京时间。
- **深度不足**：只描述了发生了什么，没分析为什么重要、影响谁、跟我军关系。→ 每条的「小马解读」必须3-5句以上。
- **文化内容敷衍**：爵士只写一个专辑名就完了。→ Kali要的是读完原文、用自己的话解释声音和氛围。
- **重复昨日内容**：昨天写过的新闻今天又写一遍。→ 写完后跟上次简报全文对比查重。
- **格式不对**：标题不统一、没有L1源标签、没有置信度字段。→ 严格按照上方格式模板。
- **逻辑链缺失**：只给出结果没有推理过程。→ 用「推演链」展示思考路径。
- **Cron执行失败（模型/Provider未指定）**：cron job 没有指定 model+provider 时会继承主配置。如果主配置的 provider 限流(429)或模型名无效(404)，整个cron会静默失败。→ 必须在cron job中显式指定 `model` 和 `provider`，不要依赖继承。
- **主题单一化（用户最频繁纠正的新痛点！）**：连续多天核心叙事被同一地缘事件霸占（如伊朗战争连续3天），导致"同样的新闻内容在日报里重复了三遍"。→ 铁律：地缘事件不超2天核心叙事，第3天让位；最高优先级信号至少3个领域；10个信息源含非洲/拉美/东亚。
