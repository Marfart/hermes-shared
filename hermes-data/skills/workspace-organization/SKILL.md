# 工作区文件整理与归类

## 核心原则

1. **先扫描，后行动** — 先完整分析目录结构、文件大小、重复情况，再执行任何删除/移动
2. **保护关键目录** — 系统目录、正在运行的服务目录、看门狗/监控脚本目录不可移动
3. **去重策略** — 同名+同大小视为重复，保留最新修改时间的版本，其余删除
4. **归类策略** — 按业务类别新建收纳目录，不破坏已有的合理目录结构
5. **空目录清理** — 整理后删除空目录（被占用的跳过，记录在报告中）

## 执行步骤

### Phase 1: 扫描分析

用 Python 扫描（避免 shell 命令超时）：
1. 遍历所有文件，建立 `{文件名: [(路径, mtime, size)]}` 映射
2. 按大小分组，同大小+同名 = 重复
3. 统计各目录文件数和总大小
4. 识别临时目录（.tmp*, .claude, __pycache__, node_modules 等）

排除 `node_modules`, `.git`, `__pycache__` 等不需要扫描的目录。

### Phase 2: 去重删除

- 同名+同大小 → 保留 mtime 最新的，删除其余
- 先统计数量和大小，报告给用户确认后再执行
- 直接 `os.remove()`，不走回收站（重复文件不是珍贵数据）

### Phase 3: 归类整理

按需新建收纳目录（不要过度分类）：

| 目录名 | 内容 |
|--------|------|
| `文档资料/` | PDF、DOCX、XLSX、PPTX、MD、TXT 等文档 |
| `图片素材/` | JPG、PNG、GIF、BMP、WebP 等图片 |
| `视频素材/` | MP4、AVI、MOV、MKV 等视频 |
| `安装包与压缩包/` | EXE、ZIP、RAR、7Z、TAR.GZ 等 |
| `脚本与工具/` | PY、SH、BAT、JS、SQL、DB 等脚本和小型工具 |
| `GitHub项目源码/` | 从 GitHub 克隆的项目仓库（含 .git 的目录） |

归类规则：
- 散落文件按扩展名归类到对应目录
- 子目录按内容主题归类（如 `EdgePlc/` → `文档资料/EdgePlc/`）
- 业务相关目录（报价、出货、客户资料等）保持原位不动
- 已存在的合理目录结构不要破坏

### Phase 4: 清理临时文件

删除以下临时/缓存目录：
- `.claude/` — Claude 缓存
- `.tmp.driveupload/` / `.tmp.drivedownload/` — Google Drive 同步缓存
- `__pycache__/` — Python 字节码缓存

### Phase 5: 报告

输出整理报告：
- 删除了多少重复文件，释放多少空间
- 新建了哪些收纳目录
- 哪些目录被占用无法删除（需用户手动处理）
- 最终目录结构

## 保护目录（不可移动/删除）

- 系统相关：`C:\Windows`, `C:\Program Files`, `AppData`
- 服务运行目录：看门狗脚本、监控进程、cron 相关
- 大型项目：`Hermes/`, `Obsidian/` 等已有完整结构的目录
- 业务核心：报价文件、产品规格书、客户资料等

## 注意事项

- 被 `PermissionError` 占用的目录不要强行删除，记录在报告中
- `shutil.move()` 跨分区可能失败，用 `shutil.copy2()` + `os.remove()` 替代
- 大目录（>10万文件）扫描必须用 Python，不能用 `du -sh` 等 shell 命令（会超时）
- 整理前先确认用户意图，特别是删除操作
