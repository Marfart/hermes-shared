# 技能優先鐵則 🐴✨

## 核心原則

**做任何事前，先加載相關技能，按技能步驟執行。**

這不是建議，是強制行為準則。每次違反都會浪費大量時間。

## 為什麼

### 反面教材（2026-06-18 Kali糾正）

Kali說「更新一個客戶的跟進」：
1. ❌ 我沒有先加載 `whatsapp-to-joinf-crm` 技能
2. ❌ 我沒有按技能步驟走（Step 0→Step 1→Step 2→Step 3→Step 4）
3. ❌ 我開始自己寫Python CDP WebSocket腳本
4. ❌ 我折騰了30分鐘（殺Chrome、重啟、WebSocket斷連、寫腳本）
5. ❌ 技能裡明明寫了用 `scripts/buyer-development/` 的現成CDP腳本

Kali原話：「為什麼斷了不能馬上連上，一直都讓你用CDP的現成你為什麼要搞那麼多東西呢，不是有現成的Skill嗎？」

### 正確做法

1. ✅ 先加載相關技能（`skill_view(name='whatsapp-to-joinf-crm')`）
2. ✅ 按技能步驟執行
3. ✅ 技能裡有現成腳本就直接用
4. ✅ 技能裡有依賴技能就一起加載
5. ✅ 遇到問題先看技能裡的「常見錯誤處理」

## 強制流程

```
┌─────────────────────────────┐
│  收到任務                    │
└─────────────┬───────────────┘
              ▼
┌─────────────────────────────┐
│  思考：有沒有相關技能？      │
│  - 技能列表裡搜關鍵詞        │
│  - 記憶裡有相關記錄嗎？      │
└─────────────┬───────────────┘
              ▼
┌─────────────────────────────┐
│  有技能？                    │
│  ├── 是 → 加載技能，按步驟做 │
│  └── 否 → 才自己從頭做      │
└─────────────┬───────────────┘
              ▼
┌─────────────────────────────┐
│  執行完後：                  │
│  - 有坑沒記？→ 更新技能      │
│  - 新發現？→ 存成新技能      │
└─────────────────────────────┘
```

## 檢查清單

每次開始任務前問自己：

- [ ] 我加載了相關技能嗎？
- [ ] 技能裡有現成腳本/工具嗎？
- [ ] 技能裡有依賴技能要一起加載嗎？
- [ ] 我是在按技能步驟做，還是在自己發明？
- [ ] 如果遇到問題，技能裡有常見錯誤處理嗎？
